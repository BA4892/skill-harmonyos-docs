# skill-harmonyos-docs

HarmonyOS 开发文档离线检索库，为 AI 辅助开发工具（如 Claude Code Skills、CodeArts Doer 等）提供结构化的鸿蒙开发文档快速定位能力。

## 项目简介

本项目从 HarmonyOS 官方开发者文档站点抓取并整理了 **API 23（HarmonyOS 6.0）** 的完整开发文档，涵盖 API 参考、开发指南、最佳实践三大核心分区，共计 **18,932 篇文档**，总内容量约 **363 MB**。

核心设计采用 **分类目录 + 子索引两步检索法**：AI 先读取极小的分类目录（1-3 KB）确定文档所属分类，再按需读取对应子索引精确定位，避免一次性加载大文件，显著降低 token 消耗。

## 检索方式

### 方式一：快速入口（推荐，已知具体 API 名称）

直接读取 `api/QUICK_ENTRY.md`（约 10 KB），通过 API 名称关键词映射到文档路径，一步到位。

### 方式二：两步检索法（探索式查找）

1. **读取分类目录**：根据需求读取对应的 `QUICK_INDEX.md`（1-3 KB）
   - 查 API → `api/QUICK_INDEX.md`
   - 查指南 → `guides/QUICK_INDEX.md`
   - 查实践 → `best/QUICK_INDEX.md`
2. **读取子索引**：根据分类目录确定所属分类，读取 `_index/分类名.md` 精确定位
3. **读取目标文档**：按子索引中的路径读取 `.md` 文件

### 方式三：层级导航（备选）

从 `index.md` 入口逐层浏览，适合不确定分类时的兜底方案。

## 作为 AI Skill 使用

本项目设计为 Claude Code / CodeArts Doer 等 AI 工具的 Skill 插件：

1. 将本项目放置到 Skills 目录下
2. AI 会在开发 ArkTS/HarmonyOS 应用时自动激活
3. 激活后 AI 通过两步检索法快速定位所需文档，无需联网查询

详细激活条件和工作流定义见 [SKILL.md](SKILL.md)。
